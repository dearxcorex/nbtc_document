# คู่มือวัด Frequency Occupancy — FCO & FBO

การวัด **Frequency Occupancy** คือการประเมินว่า "คลื่นความถี่ถูกใช้งานจริงมากแค่ไหน" เป็นเครื่องมือสำคัญในการบริหารสเปกตรัมของ กสทช.

---

## 1. Frequency Occupancy คืออะไร

**เปรียบเทียบ**: เหมือนนับจำนวน **ห้องที่มีคนอยู่** ในโรงแรม

- ถ้าโรงแรมมี 100 ห้อง แต่มีคนพักแค่ 20 ห้อง → occupancy = 20%
- คลื่นความถี่ก็เหมือนกัน: มีกี่ช่อง? ถูกใช้งานจริงกี่ช่อง?

Occupancy วัดได้ **2 มิติ**:

| มิติ | ชื่อ | คำถาม |
|------|------|-------|
| **เวลา** | FCO (Frequency Channel Occupancy) | ช่องนี้ถูกใช้กี่ % ของเวลา? |
| **ความถี่** | FBO (Frequency Band Occupancy) | ย่านนี้มีกี่ % ของช่องที่ถูกใช้? |

```
              ┌─── มิติเวลา (FCO)
              │    "ช่อง 100 MHz ถูกใช้ 75% ของเวลา"
              │
Occupancy ────┤
              │
              └─── มิติความถี่ (FBO)
                   "ย่าน FM มีช่องถูกใช้ 60%"
```

!!! note "ทำไมต้องวัด Occupancy"

    - **วางแผนจัดสรรคลื่น**: ย่านไหนว่าง สามารถเปิดให้ใช้เพิ่มได้
    - **ตรวจจับสถานีเถื่อน**: มีสัญญาณในช่องที่ไม่ได้จัดสรร
    - **ประเมินประสิทธิภาพ**: คลื่นที่จัดสรรไปแล้วถูกใช้งานจริงหรือไม่
    - **รายงาน ITU**: ข้อมูล occupancy เป็นส่วนหนึ่งของรายงานสากล

---

## 2. FCO — Frequency Channel Occupancy

**คำจำกัดความ**: เปอร์เซ็นต์ของเวลาที่ช่องสัญญาณ **มีสัญญาณเกิน Threshold** ในช่วงเวลาสังเกตการณ์

### สูตร

```
FCO (%) = จำนวน sample ที่สัญญาณ > Threshold  ×  100
          ─────────────────────────────────────
                  จำนวน sample ทั้งหมด
```

### วิธีวัด

```
 ระดับสัญญาณ (dBm)
  │
  │          ╱╲       ╱╲    ╱╲
  │    ╱╲   ╱  ╲     ╱  ╲  ╱  ╲
  │───╱──╲─╱────╲───╱────╲╱────╲──── Threshold (-90 dBm)
  │  ╱    ╲      ╲ ╱              
  │ ╱      ╲      ╲               
  │╱        ╲                      
  └────────────────────────────────→ เวลา
     ▲  ▲  ▲  ▲  ▲  ▲  ▲  ▲  ▲  ▲   ← sample ทุกๆ 1 วินาที
     ✗  ✓  ✓  ✓  ✗  ✓  ✓  ✓  ✓  ✗

     ✓ = เกิน threshold (occupied)     ✗ = ต่ำกว่า (vacant)
```

### ตัวอย่างคำนวณ

วัดช่อง **100.0 MHz** นาน **10 นาที** (600 samples, วัดทุก 1 วินาที):

- สัญญาณเกิน threshold = **450 samples**
- สัญญาณต่ำกว่า threshold = **150 samples**

```
FCO = 450 / 600 × 100 = 75%
```

**แปลว่า**: ช่อง 100.0 MHz ถูกใช้งาน **75% ของเวลา** ที่วัด

!!! tip "กฎง่ายๆ ในการตีความ FCO"

    | FCO | ความหมาย |
    |-----|---------|
    | 0% | ไม่มีสัญญาณเลย — ช่องว่าง |
    | 1-25% | ใช้งานน้อย / สัญญาณเป็นช่วงๆ |
    | 25-75% | ใช้งานปานกลาง |
    | 75-100% | ใช้งานหนาแน่น (เช่น FM broadcasting) |
    | 100% | ออกอากาศตลอดเวลา |

## 3. FBO — Frequency Band Occupancy

**คำจำกัดความ**: เปอร์เซ็นต์ของ **จำนวนช่องสัญญาณ** ที่ถูกใช้งาน ภายในย่านความถี่ที่กำหนด

### สูตร

```
FBO (%) = จำนวน channel ที่ occupied  ×  100
          ───────────────────────────
          จำนวน channel ทั้งหมดในย่าน
```

หรืออีกวิธี:

```
FBO (%) = ค่าเฉลี่ย FCO ของทุก channel ในย่าน
```

### ตัวอย่าง: ย่าน FM 87.5-108.0 MHz

```
ช่อง:  87.50  87.75  88.00  88.25  88.50  88.75  89.00 ...
       ┌────┐       ┌────┐ ┌────┐              ┌────┐
       │ ▓▓ │       │ ▓▓ │ │ ▓▓ │              │ ▓▓ │
       │ ▓▓ │       │ ▓▓ │ │ ▓▓ │              │ ▓▓ │
───────┴────┴───────┴────┴─┴────┴──────────────┴────┴────
       occupied     occupied occupied           occupied
              vacant               vacant vacant

ช่องทั้งหมด = 82 ช่อง (channel spacing 250 kHz)
ช่องที่ occupied = 49 ช่อง

FBO = 49 / 82 × 100 = 59.8%
```

**แปลว่า**: ย่าน FM ถูกใช้งานประมาณ **60%** ของช่องทั้งหมด

## 4. Threshold — เกณฑ์ตัดสินว่า "มีสัญญาณ"

Threshold คือ **ค่าที่สำคัญที่สุด** ในการวัด occupancy — ถ้าตั้งผิด ผลวัดจะผิดทั้งหมด

**เปรียบเทียบ**: เหมือนตั้งเส้น **ระดับน้ำท่วม** — ถ้าน้ำสูงกว่าเส้น = "ท่วม" / ต่ำกว่า = "ไม่ท่วม"

### 3 วิธีตั้ง Threshold (ตาม ITU-R SM.1880)

| วิธี | คำอธิบาย | ข้อดี | ข้อเสีย |
|------|---------|------|--------|
| **Fixed Threshold** | กำหนดค่าคงที่ เช่น -100 dBm | ง่าย ทำซ้ำได้ | ไม่ปรับตาม noise floor ที่เปลี่ยน |
| **Noise Floor + Offset** | วัด noise floor ก่อน แล้วบวก offset (เช่น +6 dB) | ปรับตามสภาพจริง | ต้องวัด noise floor ก่อน |
| **Statistical** | วิเคราะห์การกระจายทางสถิติ | แม่นยำที่สุด | ซับซ้อน ต้องใช้ซอฟต์แวร์ |

!!! abstract "แนะนำสำหรับงานภาคสนาม"

    ใช้วิธี **Noise Floor + 6 dB** เป็นมาตรฐาน:
    
    1. วัด **Noise Floor** ด้วย Trace Mode = Average (50+ sweeps)
    2. อ่านค่า เช่น noise floor = -105 dBm
    3. **Threshold = -105 + 6 = -99 dBm**
    4. สัญญาณที่ > -99 dBm = **occupied**

```
 ระดับ (dBm)
  │
  │    ╱╲         สัญญาณจริง
  │   ╱  ╲        (-80 dBm)
  │  ╱    ╲
  │ ╱      ╲
  │╱        ╲╱╲
  │──────────────── Threshold = -99 dBm  (Noise Floor + 6 dB)
  │  ╱╲╱╲╱╲╱╲╱╲╱╲  
  │──────────────── Noise Floor = -105 dBm
  │
  └──────────────→ ความถี่
```

!!! warning "ข้อผิดพลาดที่พบบ่อย"

    | ปัญหา | ผลที่ได้ | วิธีแก้ |
    |-------|---------|--------|
    | Threshold **ต่ำเกินไป** | Noise spike ถูกนับเป็นสัญญาณ → FCO สูงเกินจริง | เพิ่ม offset เป็น +10 dB |
    | Threshold **สูงเกินไป** | พลาดสัญญาณอ่อนที่มีจริง → FCO ต่ำเกินจริง | ลด offset เป็น +3 dB |
    | ไม่ได้วัด noise floor ใหม่ | สภาพ RF เปลี่ยน ค่าเดิมไม่ตรง | วัด noise floor ทุกครั้งก่อนเริ่มงาน |

## 5. การนำเสนอผลวัด

ผลวัด occupancy ควรนำเสนอ 4 รูปแบบ (ตาม ITU-R SM.1880):

### 5.1 กราฟ Occupancy vs Frequency

แสดง FCO ของแต่ละช่อง — เห็นว่าย่านไหนแน่น ย่านไหนว่าง

```
FCO%
100│█ █ █   █ █ █ █   █ █     █   █
 50│█ █ █ █ █ █ █ █ █ █ █ █ █ █ █ █
  0│█ █ █ █ █ █ █ █ █ █ █ █ █ █ █ █
   └──────────────────────────────────→ ความถี่
```

### 5.2 กราฟ Occupancy vs Time

แสดง FBO เปลี่ยนแปลงตามเวลา — เห็น peak/off-peak

```
FBO%
 80│            ████████
 60│      ██████        ████████
 40│  ████                      ████
 20│██                              ██
  0│──────────────────────────────────→ เวลา
   00:00  06:00  12:00  18:00  24:00
```

### 5.3 Spectrogram / Waterfall

แสดงทั้ง 3 มิติ (ความถี่ × เวลา × ระดับ) ในภาพเดียว — ดู pattern การใช้งาน

```
เวลา ↑
24:00│░░██░░██████░░░░██░░██████░░░░░░
18:00│░░██░░██████░░██████░░██████░░██
12:00│░░██░░██████░░██████░░██████░░██
06:00│░░██░░██████░░░░██░░░░██████░░░░
00:00│░░██░░░░████░░░░░░░░░░████░░░░░░
     └──────────────────────────────→ ความถี่
      87.5                       108.0

     █ = occupied (สัญญาณแรง)    ░ = vacant (ว่าง)
```

### 5.4 ตารางสรุป

| ย่าน | FBO เฉลี่ย | FBO สูงสุด | FBO ต่ำสุด | ช่วงเวลาที่ใช้มากสุด |
|------|-----------|-----------|-----------|---------------------|
| FM (87.5-108) | 70.7% | 82.1% | 45.3% | 06:00-22:00 |
| VHF (136-174) | 35.2% | 58.0% | 12.1% | 08:00-17:00 |
| UHF (400-470) | 22.8% | 41.5% | 8.3% | 09:00-16:00 |

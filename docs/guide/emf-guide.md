# คู่มือการตรวจวัดคลื่นแม่เหล็กไฟฟ้า (EMF)

คู่มือนี้อธิบายพื้นฐาน มาตรฐาน และวิธีการตรวจวัด EMF จากสถานีฐานโทรศัพท์มือถือ (2G/3G/4G/5G) แบบเข้าใจง่าย

เอกสารแบ่งเป็น 5 ส่วน อ่านตามลำดับ:

- **Part A** — พื้นฐาน: EMF คืออะไร ทำไมต้องวัด หน่วยที่ใช้
- **Part B** — มาตรฐาน: เกณฑ์ที่อนุญาต สูตรตัดสินผ่าน/ไม่ผ่าน
- **Part C** — ภาคสนาม: อุปกรณ์ ขั้นตอนวัด จุดวัด
- **Part D** — ตัวอย่าง: คำนวณจริง กรณีผ่าน/ไม่ผ่าน
- **Part E** — สรุปและอ้างอิง

---

# Part A: พื้นฐาน — รู้จัก EMF

## 1. EMF คืออะไร

**EMF (Electromagnetic Field)** = สนามแม่เหล็กไฟฟ้า — พลังงานที่แผ่ออกจากแหล่งกำเนิดคลื่นวิทยุ

!!! note "เปรียบเทียบง่ายๆ"

    EMF เหมือน **"แสงที่มองไม่เห็น"** ที่แผ่ออกจากเสาโทรศัพท์ ยิ่งอยู่ใกล้เสายิ่งแรง ยิ่งห่างยิ่งอ่อนลง

### EMF มีอยู่รอบตัวเราตลอดเวลา

| แหล่ง | ตัวอย่าง | กำลังส่ง |
|-------|---------|---------|
| ธรรมชาติ | สนามแม่เหล็กโลก, ฟ้าผ่า | ต่ำมาก |
| สถานีฐานมือถือ | เสาโทรศัพท์ 4G/5G | 2 - 100 วัตต์ต่อสายอากาศ |
| วิทยุ/โทรทัศน์ | เสาส่ง FM, DTV | 1,000 - 120,000 วัตต์ |
| ในบ้าน | เราเตอร์ WiFi, ไมโครเวฟ | 0.1 - 4 วัตต์ |

### EMF ประกอบด้วยอะไรบ้าง — ตัวแปรที่ต้องรู้

คลื่น EMF มี **2 ส่วน** ที่เดินทางไปด้วยกันเสมอ:

```
สายอากาศ
   🗼
    │
    │  ──── E (สนามไฟฟ้า) ขึ้น-ลง แนวตั้ง
    │  ════ H (สนามแม่เหล็ก) ซ้าย-ขวา แนวนอน
    │  ───→ ทิศทางการเดินทางของคลื่น
    │
    ▼
```

**E กับ H เหมือนอะไร:**

```
คิดว่าคลื่น EMF เหมือน "คลื่นในทะเล":

  E (สนามไฟฟ้า) = ความสูงของคลื่น (ยิ่งสูง ยิ่งแรง)
  H (สนามแม่เหล็ก) = ความกว้างของคลื่น (ยิ่งกว้าง ยิ่งแรง)
  S (Power Density) = พลังงานทั้งหมดที่คลื่นพาไป = E × H รวมกัน
```

### ตารางตัวแปรทั้งหมด

| สัญลักษณ์ | ชื่อ | หน่วย | คืออะไร | เปรียบเทียบ |
|----------|------|-------|--------|------------|
| **E** | Electric Field (สนามไฟฟ้า) | **V/m** (โวลต์ต่อเมตร) | แรงที่คลื่นกระทำต่อประจุไฟฟ้า | **แรงดันน้ำ** ในท่อ — ยิ่งแรง น้ำยิ่งพุ่ง |
| **H** | Magnetic Field (สนามแม่เหล็ก) | **A/m** (แอมแปร์ต่อเมตร) | แรงที่คลื่นกระทำต่อแม่เหล็ก | **กระแสน้ำ** — ยิ่งไหลเร็ว ยิ่งมีพลัง |
| **S** | Power Density (ความหนาแน่นกำลัง) | **W/m²** (วัตต์ต่อตารางเมตร) | พลังงานที่คลื่นส่งผ่านพื้นที่ 1 ตร.ม. | **ปริมาณน้ำ** ที่ตกลงบนพื้น — รวมทั้งแรงดันและกระแส |
| **SAR** | Specific Absorption Rate | **W/kg** (วัตต์ต่อกิโลกรัม) | อัตราที่ร่างกายดูดซับพลังงานจากคลื่น | ร่างกาย "กินพลังงาน" จากคลื่นเร็วแค่ไหน |
| **f** | Frequency (ความถี่) | **MHz** หรือ **GHz** | จำนวนรอบของคลื่นต่อวินาที | เสียงสูง-ต่ำ: f สูง = เสียงแหลม, f ต่ำ = เสียงทุ้ม |
| **λ** | Wavelength (ความยาวคลื่น) | **เมตร** | ระยะห่างระหว่างยอดคลื่น 2 ยอด | ยิ่งความถี่สูง ความยาวคลื่นยิ่งสั้น |

### ตัวแปรสัมพันธ์กันอย่างไร

**1. E กับ H → S**

```
S = E × H                  (ในเขต Far Field)

หรือ

S = E² / 377               (ถ้ารู้แค่ E)
S = H² × 377               (ถ้ารู้แค่ H)
```

!!! example "ตัวอย่าง"

    วัดได้ E = 5 V/m

    ```
    S = (5)² / 377 = 25 / 377 = 0.066 W/m² = 66 mW/m²
    ```

    นี่คือค่าที่เครื่อง Narda จะแสดง ถ้าวัดจุดเดียวกัน

**2. f กับ λ**

```
λ = 300 / f(MHz)

ตัวอย่าง:
  FM 100 MHz  → λ = 300/100 = 3.0 เมตร (คลื่นยาว)
  4G 900 MHz  → λ = 300/900 = 0.33 เมตร (33 ซม.)
  5G 3500 MHz → λ = 300/3500 = 0.086 เมตร (8.6 ซม. — คลื่นสั้น)
```

**3. ตัวเลข 377 คืออะไร**

```
377 Ω = Wave Impedance ของอากาศ

เหมือน "ความต้านทาน" ของอากาศต่อคลื่น
เป็นค่าคงที่ทางฟิสิกส์ ใช้แปลงระหว่าง V/m กับ W/m²

E / H = 377    (ในเขต Far Field)
S = E² / 377
S = H² × 377
```

### สรุปความสัมพันธ์ทั้งหมด

```
    f (ความถี่)  ←──── λ = 300/f ────→  λ (ความยาวคลื่น)
         │
         │  กำหนดค่า limit
         ▼
    E_limit, S_limit (จากตาราง ICNIRP)
         │
         │  เทียบกับค่าที่วัด
         ▼
    E (วัดได้) ←── S = E²/377 ──→ S (วัดได้)
         │                              │
         │  SAR = ร่างกายดูดซับ          │
         └──────────────────────────────┘
              ทั้งหมดเชื่อมกันหมด
```

## 2. ทำไมต้องวัด EMF

คลื่น EMF ที่แรงเกินไปอาจส่งผลต่อร่างกาย:

| ผลระยะสั้น | ผลระยะยาว (ยังศึกษาอยู่) |
|-----------|------------------------|
| ความร้อนในเนื้อเยื่อ | ระบบภูมิคุ้มกันอ่อนแอ |
| ผลต่อเครื่องกระตุ้นหัวใจ | ความเสี่ยงมะเร็ง |
| ต้อกระจกตา | การเติบโตของเซลล์ผิดปกติ |

!!! warning "IARC จัดให้ EMF อยู่ในกลุ่ม 2B"

    องค์การวิจัยมะเร็ง (IARC/WHO) จัดให้คลื่น EMF ความถี่วิทยุเป็น **"อาจเป็นสารก่อมะเร็ง" (Group 2B)** — มีหลักฐานจำกัด อยู่กลุ่มเดียวกับไอเสียเบนซิน

### ร่างกายดูดซับ EMF ไม่เท่ากันทุกความถี่

- **30 - 100 MHz**: ดูดซับสูงสุด (ความสูงร่างกาย ≈ ครึ่งความยาวคลื่น)
- คนสูง 1.80 ม. → resonance ที่ **83.3 MHz**
- อวัยวะดูดซับสูง: ตา (+++) สมอง (++) เข่า (+++) ฟันอุด (+++)

**ดังนั้น** จึงต้องวัด EMF เพื่อยืนยันว่าค่าที่ประชาชนได้รับ **ต่ำกว่าเกณฑ์ปลอดภัย**

## 3. หน่วยที่ใช้วัด: V/m กับ W/m²

การวัด EMF มี **2 หน่วย** ที่ใช้ ขึ้นอยู่กับเครื่องวัด:

| หน่วย | ชื่อ | วัดอะไร | เครื่องที่ใช้ |
|-------|------|--------|-------------|
| **V/m** | โวลต์ต่อเมตร | ความแรงสนามไฟฟ้า | Spectrum Analyzer (R&S FSH8) |
| **W/m²** | วัตต์ต่อตารางเมตร | ความหนาแน่นกำลัง | Narda SRM-3006 / NBM-550 |

### เปรียบเทียบง่ายๆ

```
สายยางรดน้ำ:
  V/m  = แรงดันน้ำ (ฉีดแรงแค่ไหน)
  W/m² = ปริมาณน้ำที่ตกลงบนพื้น 1 ตร.ม. (เปียกแค่ไหน)
```

ทั้งสอง **วัดสิ่งเดียวกัน** แค่มุมมองต่างกัน สัมพันธ์กันผ่านสูตร:

```
W/m² = (V/m)² ÷ 377
```

### ตารางแปลงค่าด่วน

| V/m | W/m² | mW/m² |
|-----|------|-------|
| 1 | 0.003 | 2.7 |
| 5 | 0.066 | 66 |
| 10 | 0.265 | 265 |
| 28 | 2.08 | 2,080 |
| 41.25 | 4.51 | 4,513 |
| 61 | 9.87 | 9,872 |

### หน่วยที่เครื่อง Narda อาจแสดง

| แสดงเป็น | แปลงเป็น W/m² |
|---------|--------------|
| **W/m²** | ใช้ได้เลย |
| **mW/m²** | หาร 1,000 |
| **µW/m²** | หาร 1,000,000 |
| **mW/cm²** | คูณ 10 |

**ตัวอย่าง**: Narda อ่านได้ 450 µW/m² = 0.00045 W/m² = **ต่ำมาก ผ่านสบาย**

# Part B: มาตรฐานและเกณฑ์

## 4. มาตรฐานที่ใช้

### ICNIRP 2020 — มาตรฐานสากล

**ICNIRP** (International Commission on Non-Ionizing Radiation Protection) เป็นองค์กรอิสระทำงานร่วมกับ WHO

- ใช้ใน **135 ประเทศ** ทั่วโลก
- ค่าจำกัดต่ำกว่า threshold ที่เป็นอันตราย **50 เท่า** (safety factor)

### มาตรฐานไทย: กทช. มท. 5001-2550

ประเทศไทยใช้มาตรฐาน **กทช. มท. 5001-2550** ซึ่งอ้างอิงจาก ICNIRP ครอบคลุมย่าน **9 kHz - 300 GHz**

---

## 5. ค่า limit มาจากไหน — คิดยังไง

ก่อนจะดูตัวเลขในตาราง ต้องเข้าใจก่อนว่า **ค่า limit ไม่ได้คิดขึ้นมาลอยๆ** แต่มาจากการทดลองทางวิทยาศาสตร์ 3 ขั้นตอน:

### ขั้นที่ 1: หาจุดที่เริ่มเป็นอันตราย (Threshold)

นักวิทยาศาสตร์ทดลองในห้อง Lab ว่า คลื่น EMF ต้องแรงแค่ไหนถึงทำให้ **อุณหภูมิร่างกายสูงขึ้น 1°C** (จุดที่เริ่มมีผลต่อสุขภาพ)

ผลทดลอง: ต้องใช้พลังงาน **SAR = 4 W/kg** (วัตต์ต่อกิโลกรัม) ถึงจะทำให้ร่างกายร้อนขึ้น 1°C

```
SAR = Specific Absorption Rate
    = อัตราที่ร่างกายดูดซับพลังงานจากคลื่น
    = 4 W/kg คือ "จุดเริ่มอันตราย"
```

!!! example "เปรียบเทียบง่ายๆ"

    สมมติคุณใช้ไมโครเวฟอุ่นน้ำ 1 แก้ว — ถ้าคลื่นแรง 4 W/kg ร่างกายคุณจะร้อนขึ้นเหมือนน้ำในไมโครเวฟ (แต่แค่ 1 องศา)

    **จุดนี้คือจุดที่ "เริ่ม" มีผล** — ยังไม่อันตรายร้ายแรง แต่เราไม่อยากให้ถึงจุดนี้

### ขั้นที่ 2: หารด้วย Safety Factor (เผื่อความปลอดภัย)

ICNIRP ไม่ได้ตั้งเกณฑ์ที่จุดอันตรายพอดี แต่ **หารลง** เพื่อเผื่อความปลอดภัย:

```
จุดอันตราย (Threshold)     = SAR 4 W/kg
                              │
    ÷ 10 (คนทำงาน)           │    ÷ 50 (ประชาชน)
         │                    │         │
         ▼                              ▼
    SAR 0.4 W/kg                   SAR 0.08 W/kg
    (Occupational)                 (General Public)
```

!!! example "เปรียบเทียบง่ายๆ"

    สมมติสะพานรับน้ำหนักได้ 50 ตัน (= Threshold)

    - **คนทำงาน** (รู้ความเสี่ยง มีอุปกรณ์ป้องกัน): อนุญาต 5 ตัน (÷10)
    - **ประชาชน** (เด็ก คนชรา คนท้อง ไม่รู้ความเสี่ยง): อนุญาต **1 ตัน** (÷50)

    เราตรวจวัดให้ประชาชน → ใช้ค่าที่ **เข้มงวดกว่า 50 เท่า** จากจุดอันตราย

| กลุ่ม | Safety Factor | SAR limit | เทียบกับ Threshold |
|-------|--------------|-----------|-------------------|
| จุดอันตราย (Threshold) | - | 4 W/kg | 100% |
| **คนทำงาน** (Occupational) | ÷10 | 0.4 W/kg | 10% |
| **ประชาชน** (General Public) | ÷50 | 0.08 W/kg | **2%** |

### ขั้นที่ 3: แปลง SAR → V/m และ W/m²

SAR วัดโดยตรงไม่ได้ในภาคสนาม (ต้องใช้หุ่นจำลองใน Lab) จึงต้อง **แปลงเป็นค่าที่วัดได้** คือ V/m และ W/m²

ICNIRP ใช้แบบจำลองร่างกายมนุษย์ (computational model) คำนวณว่า:

"ถ้า SAR ต้องไม่เกิน 0.08 W/kg → ค่า E-field ที่จุดนั้นต้องไม่เกินเท่าไร"

ผลลัพธ์คือ **ตารางค่า limit** ที่เราใช้:

```
SAR 0.08 W/kg (ประชาชน)
      │
      │  แปลงด้วยแบบจำลองร่างกาย
      │  (คิดกรณีเลวร้ายสุด worst-case)
      ▼
┌──────────────────────────────────┐
│  ย่าน 10-400 MHz:  28 V/m       │
│  ย่าน 400-2000 MHz: 1.375√f V/m │  ← ตารางที่เราใช้!
│  ย่าน 2-300 GHz:   61 V/m       │
└──────────────────────────────────┘
```

### ทำไมค่า limit เปลี่ยนตามความถี่

เพราะร่างกาย **ดูดซับคลื่นไม่เท่ากัน** ในแต่ละความถี่:

```
    ▲ การดูดซับ
    │
    │        ╱╲
    │       ╱  ╲           ← ย่าน 30-100 MHz ดูดซับสูงสุด
    │      ╱    ╲             (resonance กับร่างกาย)
    │     ╱      ╲            → limit ต้องเข้มงวด (28 V/m)
    │    ╱        ╲
    │   ╱          ╲──────── ← ย่าน 2+ GHz ดูดซับน้อยลง
    │  ╱                       → limit ผ่อนคลายได้ (61 V/m)
    │ ╱
    └──────────────────────► ความถี่ (MHz)
      30   100  400  2000
```

| ย่านความถี่ | ร่างกายดูดซับ | limit เข้มงวดแค่ไหน |
|------------|-------------|-------------------|
| 10-400 MHz (FM, DTV) | **สูงมาก** (resonance) | เข้มงวดสุด: **28 V/m** / **2 W/m²** |
| 400-2000 MHz (4G) | ปานกลาง | กลาง: **41-61 V/m** / **4.5-10 W/m²** |
| 2-300 GHz (5G, WiFi) | ต่ำ (ดูดซับแค่ผิวหนัง) | ผ่อนคลาย: **61 V/m** / **10 W/m²** |

### สรุปภาพรวมทั้งหมด

```
ทดลองใน Lab
  "คลื่นแรงแค่ไหนถึงร้อนขึ้น 1°C"
  → SAR 4 W/kg = จุดอันตราย
         │
         │  ÷50 (เผื่อให้ประชาชน)
         ▼
  SAR 0.08 W/kg = เกณฑ์ประชาชน
         │
         │  แปลงด้วยแบบจำลองร่างกาย
         ▼
  ตาราง limit (V/m, W/m²) ← ค่าที่เราเอาไปใช้ตรวจ
         │
         │  ต่ำกว่า limit อีก ~50 เท่า
         ▼
  ค่าจริงที่วัดได้ในไทย ← ปลอดภัยมาก (~2,500 เท่าจากจุดอันตราย)
```

## 6. ค่าขีดจำกัดสำหรับประชาชนทั่วไป

### ตาราง E-field (V/m)

| ย่านความถี่ | E_limit (V/m) |
|------------|--------------|
| 10 MHz - 400 MHz | **28** |
| 400 MHz - 2 GHz | **1.375√f** |
| 2 GHz - 300 GHz | **61** |

*(f = ความถี่ใน MHz)*

### ตาราง Power Density (W/m²) — สำหรับเครื่อง Narda

| ย่านความถี่ | S_limit (W/m²) | ตัวอย่างบริการ |
|------------|---------------|---------------|
| 10 MHz - 400 MHz | **2** | FM Radio, DTV |
| 400 MHz - 2 GHz | **f/200** | 4G มือถือ |
| 2 GHz - 300 GHz | **10** | 5G, WiFi |

### ค่าที่ใช้บ่อยในงาน กสทช.

| ความถี่ | บริการ | E_limit (V/m) | S_limit (W/m²) |
|--------|--------|--------------|---------------|
| 100 MHz | FM Radio | 28 | 2 |
| 700 MHz | 4G/5G | 36.40 | 3.5 |
| 900 MHz | 4G | 41.25 | 4.5 |
| 1800 MHz | 4G | 58.36 | 9.0 |
| 2100 MHz | 4G | 61 | 10 |
| 2600 MHz | 5G | 61 | 10 |
| 3500 MHz | 5G (n78) | 61 | 10 |

!!! tip "วิธีคิดย่าน 400-2000 MHz"

    - **V/m**: E_limit = 1.375 × √f เช่น 900 MHz → 1.375 × 30 = 41.25
    - **W/m²**: S_limit = f ÷ 200 เช่น 900 MHz → 900/200 = 4.5

### E_limit กับ S_limit คืออะไร

| คำ | ความหมาย |
|----|---------|
| **E_limit** | ค่า V/m **สูงสุดที่อนุญาต** ตามตารางข้างบน |
| **S_limit** | ค่า W/m² **สูงสุดที่อนุญาต** ตามตารางข้างบน |

ค่าที่**วัดได้จริง**ในพื้นที่ต้อง**ต่ำกว่า**ค่า limit เหล่านี้

## 7. สูตรตัดสิน ผ่าน/ไม่ผ่าน

### กรณีวัดความถี่เดียว

เปรียบเทียบค่าที่วัดได้กับค่า limit ตรงๆ:

```
ถ้า วัดได้ < limit → ผ่าน
ถ้า วัดได้ > limit → ไม่ผ่าน
```

### กรณีหลายความถี่พร้อมกัน (Multi-source Summation)

เมื่อมีสถานีฐานหลายค่ายบนเสาเดียวกัน ต้อง**รวมผลทุกความถี่**:

**สูตรที่ใช้ในงาน (ย่อ — เฉพาะย่าน >1 MHz):**

```
แบบ V/m:   Σ (Eᵢ / E_limit,i)² ≤ 1
แบบ W/m²:  Σ (Sᵢ / S_limit,i)  ≤ 1
```

!!! note "ทำไมสูตร V/m ต้องยกกำลังสอง แต่ W/m² ไม่ต้อง"

    เพราะ W/m² = (V/m)² ÷ 377 — **ยกกำลังสองมาแล้ว** ในตัว

    ทั้งสองสูตรให้ **ผลเท่ากัน** เลือกใช้ตามเครื่องที่มี

### สูตร ICNIRP 2020 แบบเต็ม vs แบบที่เราใช้

**เปรียบเทียบ**: สูตรเต็มเหมือน **กฎหมายฉบับเต็ม** ที่ครอบคลุมทุกกรณี — แต่งานเราใช้แค่ **มาตราเดียว** ที่เกี่ยวข้อง

#### ทำไมมี 2 แบบ

สูตร ICNIRP 2020 แบ่งความถี่เป็น **2 ย่าน** เพราะร่างกายตอบสนองต่างกัน:

```
◄──── ย่านที่ 1 ────►◄──────── ย่านที่ 2 ────────────►
    100 kHz - 1 MHz           1 MHz - 300 GHz

  ผลกระทบ: กระตุ้น          ผลกระทบ: ความร้อน
  เส้นประสาท/กล้ามเนื้อ      ในเนื้อเยื่อ

  ใช้ค่าคงที่ c, d           ใช้ค่า limit จากตาราง
  (คำนวณต่างกัน)            (หัวข้อ 6)
```

#### สูตรเต็ม (ไม่ต้องจำ — ดูเพื่อเข้าใจภาพรวม)

```
   ย่านที่ 1 (ต่ำกว่า 1 MHz)     ย่านที่ 2 (สูงกว่า 1 MHz)
  ┌────────────────────────┐  ┌────────────────────────┐
  │                        │  │                        │
  │  Σ (ค่าวัด / ค่าคงที่)² │ +│  Σ (ค่าวัด / ค่า limit)²│ ≤ 1
  │                        │  │                        │
  └────────────────────────┘  └────────────────────────┘
         ↑                              ↑
    เราไม่ได้วัดย่านนี้           เราวัดย่านนี้!
    (= 0 ตัดทิ้งได้)
```

#### สูตรที่เราใช้จริง (จำแค่นี้!)

งาน กสทช. วัดแค่ FM (87-108 MHz) และสถานีฐาน (700-3500 MHz) — ทั้งหมดอยู่ **ย่านที่ 2 (>1 MHz)** → ย่านที่ 1 เป็น 0 ตัดทิ้งได้เลย

```
แบบ V/m (Spectrum Analyzer):

    (E₁/E_limit₁)² + (E₂/E_limit₂)² + ... ≤ 1

แบบ W/m² (Narda):

    (S₁/S_limit₁) + (S₂/S_limit₂) + ... ≤ 1
```

!!! tip "สรุปสั้นๆ"

    - **สูตรเต็ม** = 2 ท่อน (ย่านต่ำ + ย่านสูง)
    - **งานเรา** = ใช้แค่ท่อนย่านสูง (>1 MHz)
    - **จำแค่**: รวม (ค่าวัด ÷ ค่า limit) ของทุกความถี่ → ผลรวม ≤ 1 = **ผ่าน**

!!! example "ตัวอย่างให้เห็นภาพ"

    วัดได้ FM 100 MHz และ 4G 900 MHz:

    ```
    สูตรเต็ม:
    [ย่าน <1 MHz = ไม่มี = 0] + [(E_FM/28)² + (E_4G/41.25)²] ≤ 1
                                  └──────────────────────────┘
                                    ↑ เหลือแค่ส่วนนี้

    สูตรที่เราใช้:
    (E_FM / 28)² + (E_4G / 41.25)² ≤ 1
    ```

## 9. Near Field vs Far Field (ต้องรู้ก่อนวัด)

สนาม EMF แบ่งเป็น 2 โซน ตามระยะห่างจากสายอากาศ:

| โซน | ระยะ | ลักษณะ |
|-----|------|--------|
| **Near Field** | ใกล้สายอากาศ | E กับ H ไม่สัมพันธ์กัน ต้องวัดทั้งสอง |
| **Far Field** | ไกลออกไป | E กับ H สัมพันธ์กัน วัดอย่างเดียวพอ |

**สูตรแบ่งเขต**: Far Field เริ่มที่ R = 2D²/λ (D = ขนาดสายอากาศ, λ = ความยาวคลื่น)

| ความถี่ | บริการ | Far Field เริ่มที่ (สายอากาศ 1 ม.) |
|--------|--------|----------------------------------|
| 100 MHz | FM | 0.67 ม. |
| 900 MHz | 4G | 6.0 ม. |
| 2,450 MHz | WiFi | 16.3 ม. |
| 3,500 MHz | 5G | 23.3 ม. |

!!! info "ในทางปฏิบัติ"

    เราวัดที่ระยะ **50 ม. ขึ้นไป** จากสถานีฐาน → **อยู่ใน Far Field เสมอ** → วัดแค่ค่าเดียว (E หรือ S) แล้วคำนวณที่เหลือได้

# Part D: ตัวอย่างการคำนวณ

## 11. Single Source — ความถี่เดียว (เริ่มจากง่ายสุด)

กรณีที่ง่ายที่สุดคือ มีสถานีเดียว ความถี่เดียว แค่เทียบค่าตรงๆ

### วิธีคิด

```
ค่าที่วัดได้  <  ค่า limit  →  ✅ ผ่าน
ค่าที่วัดได้  >  ค่า limit  →  ❌ ไม่ผ่าน
```

### ตัวอย่าง 1A: วัดด้วย Narda (W/m²) — สถานี FM 100 MHz

```
ขั้น 1: Narda อ่านได้ = 0.85 W/m²

ขั้น 2: หา limit → 100 MHz อยู่ย่าน 10-400 MHz → S_limit = 2 W/m²

ขั้น 3: เทียบ → 0.85 < 2 → ✅ ผ่าน

ขั้น 4: คิดเปอร์เซ็นต์ → (ค่าวัด / limit) × 100
        = (0.85 / 2) × 100
        = 0.425 × 100
        = 42.5%

  Narda อ่านได้   เกณฑ์
  ───────────────────────────────
  ████████░░░░░░░░░░░░░░░░░░░░░░
  0.85 W/m²              2 W/m²
  (42.5%)                (100%)
```

### ตัวอย่าง 1B: วัดด้วย Narda — สถานี 4G 900 MHz

```
ขั้น 1: Narda อ่านได้ = 120 mW/m²
        แปลงหน่วย: 120 ÷ 1000 = 0.12 W/m²

ขั้น 2: หา limit → 900 MHz อยู่ย่าน 400-2000 MHz
        S_limit = f / 200 = 900 / 200 = 4.5 W/m²

ขั้น 3: เทียบ → 0.12 < 4.5 → ✅ ผ่าน

ขั้น 4: คิดเปอร์เซ็นต์ → (ค่าวัด / limit) × 100
        = (0.12 / 4.5) × 100
        = 0.0267 × 100
        = 2.67% ≈ 2.7%

  Narda อ่านได้   เกณฑ์
  ───────────────────────────────────────────────
  ██░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░
  0.12 W/m²                             4.5 W/m²
  (2.7%)                                 (100%)
```

### ตัวอย่าง 1C: วัดด้วย Spectrum Analyzer (V/m) — FM 100 MHz

```
ขั้น 1: Spectrum Analyzer อ่านได้ = 27.7 V/m

ขั้น 2: หา limit → 100 MHz อยู่ย่าน 10-400 MHz → E_limit = 28 V/m

ขั้น 3: เทียบ → 27.7 < 28 → ✅ ผ่าน

ขั้น 4: คิดเปอร์เซ็นต์ → (ค่าวัด / limit) × 100
        = (27.7 / 28) × 100
        = 0.9893 × 100
        = 98.9% ← เฉียดมาก!

  ลองตรวจสอบด้วย W/m²:
  แปลงค่าวัด: S = (27.7)² / 377 = 767.29 / 377 = 2.035 W/m²
  limit:      S_limit = 2 W/m²
  เปอร์เซ็นต์: (2.035 / 2) × 100 = 101.75% → ❌ เกินเกณฑ์!
```

!!! warning "ทำไมผลต่างกัน?"

    V/m เฉียดผ่าน (98.9%) แต่ W/m² ไม่ผ่าน (101.75%)

    เพราะ V/m limit (28) ถูก**ปัดเศษ**ขึ้นจากค่าจริง (27.74 V/m ที่ S=2 W/m² พอดี)

    **ในทางปฏิบัติ**: ถ้าค่าใกล้เกณฑ์มาก (>90%) ให้ถือว่า **เฉียดไม่ผ่าน** และรายงานตามค่าจริง

### สรุป Single Source

```
ขั้น 1: อ่านค่าจากเครื่อง
         │
ขั้น 2: หา limit ของความถี่นั้นจากตาราง
         │
ขั้น 3: เทียบ
         ├── วัดได้ < limit → ✅ ผ่าน
         └── วัดได้ > limit → ❌ ไม่ผ่าน
```

**ง่ายมาก!** แต่ในความเป็นจริง เสา 1 ต้นมักมีหลายค่าย หลายความถี่ → ต้องใช้ Multi-source

## 12. Multi-source — หลายความถี่พร้อมกัน (สถานการณ์จริง)

### ทำไมต้อง Multi-source — อธิบายทีละขั้น

!!! info "ปัญหา"

    เสาโทรศัพท์ 1 ต้น มักมีหลายค่าย หลายความถี่ ติดอยู่ด้วยกัน เช่น AIS 900 MHz + AIS 2100 MHz
    แต่ละความถี่มี**เกณฑ์ไม่เท่ากัน** → เทียบค่าเดียวไม่ได้ ต้อง**รวมผลทุกความถี่**

**เปรียบเทียบง่ายๆ:**

```
สมมติคุณมีถังน้ำ 1 ถัง (= เกณฑ์ 100%)

  แต่ละค่ายเทน้ำใส่ถังคนละนิด:
  AIS 900    เท   1.5%  ████
  AIS 2100   เท   1.7%  █████
                        ─────
  รวม              3.2%  █████████

  ถังยังว่างอีก 96.8% → ผ่านสบาย ✅

  แต่ถ้าเทจนล้น (รวม > 100%) → ไม่ผ่าน ❌
```

---

### ตัวอย่าง 2: แสดงการคำนวณทุกขั้นตอน

สมมติวัดที่เสาร่วม AIS มี 2 ความถี่: 900 MHz และ 2100 MHz

---

**ขั้นที่ 1: อ่านค่าจากเครื่อง**

เครื่อง Narda อ่านได้:

- 900 MHz → **66.4 mW/m²**
- 2100 MHz → **169.8 mW/m²**

Spectrum Analyzer อ่านได้:

- 900 MHz → **5.0 V/m**
- 2100 MHz → **8.0 V/m**

---

**ขั้นที่ 2: หาค่า limit ของแต่ละความถี่**

ดูจากตารางในหัวข้อ 5:

- 900 MHz อยู่ย่าน 400-2000 MHz
    - E_limit = 1.375 × √900 = 1.375 × 30 = **41.25 V/m**
    - S_limit = 900 ÷ 200 = **4.5 W/m²**
- 2100 MHz อยู่ย่าน 2-300 GHz
    - E_limit = **61 V/m**
    - S_limit = **10 W/m²**

---

**ขั้นที่ 3: คำนวณอัตราส่วนของแต่ละความถี่**

**แบบ Narda (W/m²) — ง่ายกว่า แค่หารเฉยๆ:**

```
แปลงหน่วยก่อน: mW/m² → W/m² (หาร 1000)
  900 MHz:  66.4 mW/m² = 0.0664 W/m²
  2100 MHz: 169.8 mW/m² = 0.1698 W/m²

คำนวณอัตราส่วน:
  900 MHz:  S / S_limit = 0.0664 / 4.5 = 0.0148  (ใช้ 1.48%)
  2100 MHz: S / S_limit = 0.1698 / 10  = 0.0170  (ใช้ 1.70%)
```

**แบบ V/m — ต้องยกกำลังสอง:**

```
คำนวณอัตราส่วน:
  900 MHz:  (E / E_limit)² = (5.0 / 41.25)² = (0.1212)² = 0.0147  (ใช้ 1.47%)
  2100 MHz: (E / E_limit)² = (8.0 / 61)²    = (0.1311)² = 0.0172  (ใช้ 1.72%)
```

---

**ขั้นที่ 4: บวกรวมทุกความถี่**

**แบบ Narda:**

```
ผลรวม = 0.0148 + 0.0170 = 0.0318
```

**แบบ V/m:**

```
ผลรวม = 0.0147 + 0.0172 = 0.0319
```

---

**ขั้นที่ 5: ตัดสิน**

```
0.032 ≤ 1 → ✅ ผ่าน (ใช้แค่ 3.2% ของเกณฑ์ที่อนุญาต)
```

!!! tip "สังเกต"

    ทั้งสองสูตรได้ผล **~0.032 เท่ากัน** ใช้สูตรไหนก็ได้ตามเครื่องที่มี

### สรุปขั้นตอน Multi-source

```
ขั้น 1: อ่านค่าจากเครื่อง (V/m หรือ W/m²)
         │
ขั้น 2: หา limit ของแต่ละความถี่จากตาราง
         │
ขั้น 3: คำนวณอัตราส่วน = ค่าวัด / limit
         │  (ถ้า V/m ต้องยกกำลังสอง)
         │  (ถ้า W/m² ไม่ต้อง)
         │
ขั้น 4: บวกรวมทุกความถี่
         │
ขั้น 5: ตัดสิน
         ├── ≤ 1 → ✅ ผ่าน
         └── > 1 → ❌ ไม่ผ่าน
```

### ตัวอย่าง 3: เสาร่วม 4 ค่าย — Narda วัดได้ (W/m²)

**ขั้น 1: Narda อ่านค่าแต่ละความถี่ (mW/m²) แล้วแปลงเป็น W/m²**

```
AIS  900 MHz:  Narda อ่านได้ 27.2 mW/m²  → 27.2 ÷ 1000 = 0.0272 W/m²
AIS  2100 MHz: Narda อ่านได้ 53.7 mW/m²  → 53.7 ÷ 1000 = 0.0537 W/m²
TRUE 850 MHz:  Narda อ่านได้ 20.8 mW/m²  → 20.8 ÷ 1000 = 0.0208 W/m²
TRUE 2600 MHz: Narda อ่านได้ 69.0 mW/m²  → 69.0 ÷ 1000 = 0.0690 W/m²
DTAC 1800 MHz: Narda อ่านได้ 40.3 mW/m²  → 40.3 ÷ 1000 = 0.0403 W/m²
NT   700 MHz:  Narda อ่านได้ 5.97 mW/m²  → 5.97 ÷ 1000 = 0.0060 W/m²
```

**ขั้น 2: หา S_limit ของแต่ละความถี่**

```
AIS  900 MHz:  ย่าน 400-2000 → S_limit = 900/200  = 4.5 W/m²
AIS  2100 MHz: ย่าน 2-300 GHz → S_limit             = 10 W/m²
TRUE 850 MHz:  ย่าน 400-2000 → S_limit = 850/200  = 4.25 W/m²
TRUE 2600 MHz: ย่าน 2-300 GHz → S_limit             = 10 W/m²
DTAC 1800 MHz: ย่าน 400-2000 → S_limit = 1800/200 = 9.0 W/m²
NT   700 MHz:  ย่าน 400-2000 → S_limit = 700/200  = 3.5 W/m²
```

**ขั้น 3: คำนวณอัตราส่วน S/S_limit ของแต่ละความถี่ แล้วคิด %**

```
AIS  900:  0.0272 / 4.5  = 0.0060  → 0.0060 × 100 = 0.60%
AIS  2100: 0.0537 / 10   = 0.0054  → 0.0054 × 100 = 0.54%
TRUE 850:  0.0208 / 4.25 = 0.0049  → 0.0049 × 100 = 0.49%
TRUE 2600: 0.0690 / 10   = 0.0069  → 0.0069 × 100 = 0.69%
DTAC 1800: 0.0403 / 9.0  = 0.0045  → 0.0045 × 100 = 0.45%
NT   700:  0.0060 / 3.5  = 0.0017  → 0.0017 × 100 = 0.17%
```

**ขั้น 4: บวกรวม**

```
ผลรวม = 0.0060 + 0.0054 + 0.0049 + 0.0069 + 0.0045 + 0.0017
      = 0.0294

คิดเป็น % = 0.0294 × 100 = 2.94%
```

**ขั้น 5: ตัดสิน**

```
0.0294 ≤ 1 → ✅ ผ่าน (ใช้แค่ 2.94% ของเกณฑ์)

  ◄────────── เกณฑ์ 1.0 (100%) ────────────────────────────────►
  ████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░
  2.94%                                                   100%
  ✅ ผ่าน — ถังน้ำแทบไม่มีน้ำเลย
```

### ตัวอย่าง 4: เสาร่วมที่มี FM Radio — Narda (W/m²)

**ขั้น 1-2: Narda อ่านค่า + หา limit**

```
FM สถานี A  100 MHz:    Narda = 0.382 W/m²  limit: ย่าน 10-400 → S_limit = 2 W/m²
FM สถานี B  104.25 MHz: Narda = 0.192 W/m²  limit: ย่าน 10-400 → S_limit = 2 W/m²
AIS 4G      900 MHz:    Narda = 0.012 W/m²  limit: 900/200     → S_limit = 4.5 W/m²
TRUE 4G     2100 MHz:   Narda = 0.024 W/m²  limit: ย่าน >2 GHz → S_limit = 10 W/m²
```

**ขั้น 3: คำนวณอัตราส่วน + %**

```
FM A:   0.382 / 2   = 0.191  → 0.191 × 100 = 19.1%  ← FM กินเยอะ!
FM B:   0.192 / 2   = 0.096  → 0.096 × 100 =  9.6%  ← FM กินเยอะ!
AIS:    0.012 / 4.5 = 0.003  → 0.003 × 100 =  0.3%
TRUE:   0.024 / 10  = 0.002  → 0.002 × 100 =  0.2%
```

**ขั้น 4: บวกรวม**

```
ผลรวม = 0.191 + 0.096 + 0.003 + 0.002 = 0.292
คิดเป็น % = 0.292 × 100 = 29.2%
```

**ขั้น 5: ตัดสิน**

```
0.292 ≤ 1 → ✅ ผ่าน (ใช้ 29.2% ของเกณฑ์)

  แต่สังเกต:
  FM A  ████████████████████░░░░░░░░░░  19.1%
  FM B  ██████████░░░░░░░░░░░░░░░░░░░░   9.6%
  AIS   █░░░░░░░░░░░░░░░░░░░░░░░░░░░░░   0.3%
  TRUE  ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░   0.2%
        ──────────────────────────────
```

### ตัวอย่าง 5: กรณีไม่ผ่าน — FM กำลังสูง

**ขั้น 1-2: Narda อ่านค่า + หา limit**

```
FM สถานี A  95 MHz:  Narda = 1.284 W/m²  limit: ย่าน 10-400 → S_limit = 2 W/m²
FM สถานี B  100 MHz: Narda = 0.860 W/m²  limit: ย่าน 10-400 → S_limit = 2 W/m²
FM สถานี C  107 MHz: Narda = 0.066 W/m²  limit: ย่าน 10-400 → S_limit = 2 W/m²
```

**ขั้น 3: คำนวณอัตราส่วน + %**

```
FM A: 1.284 / 2 = 0.642  → 0.642 × 100 = 64.2%  ← สถานีเดียวกินเกินครึ่ง!
FM B: 0.860 / 2 = 0.430  → 0.430 × 100 = 43.0%
FM C: 0.066 / 2 = 0.033  → 0.033 × 100 =  3.3%
```

**ขั้น 4: บวกรวม**

```
ผลรวม = 0.642 + 0.430 + 0.033 = 1.105
คิดเป็น % = 1.105 × 100 = 110.5%  ← เกิน 100%!
```

**ขั้น 5: ตัดสิน**

```
1.105 > 1 → ❌ ไม่ผ่าน! (เกินเกณฑ์ 10.5%)

  ◄────────── เกณฑ์ 1.0 (100%) ───────────────────►│OVER│
  ██████████████████████████████████████████████████│████│
  64.2%          43.0%              3.3%           │10.5%│
  ❌ ไม่ผ่าน — เกินเกณฑ์ 10.5%
```

**ต้องทำอย่างไร:**

1. ตรวจสอบว่าสถานี FM ส่งกำลังเกินใบอนุญาตหรือไม่
2. แจ้งสถานีให้ลดกำลังส่ง หรือปรับทิศสายอากาศ
3. วัดซ้ำหลังแก้ไข

## 13. สรุปแบบเข้าใจง่าย

| คำถาม | คำตอบ |

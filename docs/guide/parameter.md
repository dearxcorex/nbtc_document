# คู่มือตรวจ FM

อธิบายค่าพารามิเตอร์ที่ใช้ในการตรวจสอบมาตรฐานทางเทคนิคของสถานีวิทยุกระจายเสียงระบบ FM ตามมาตรฐาน กสทช. และอ้างอิง ITU-R

!!! abstract "คู่มือการตรวจสอบการใช้งานคลื่นความถี่กิจการกระจายเสียงระบบเอฟเอ็ม"

    เอกสารอ้างอิงจาก สำนักวิศวกรรมและเทคโนโลยีในกิจการกระจายเสียงและโทรทัศน์ (ทส.) และสำนักรับรองมาตรฐานวิศวกรรมในกิจการกระจายเสียงและโทรทัศน์ (รส.)

    ครอบคลุมเรื่อง: คุณลักษณะทางเทคนิคสถานี, มาตรฐานเครื่องส่ง, การคำนวณ ERP, พื้นที่ให้บริการ, การตรวจสอบการรบกวน

    [:material-file-pdf-box: ดาวน์โหลด PDF](../assets/fm-inspection-manual.pdf){ .md-button }

## 1. Occupied Bandwidth (แบนด์วิดท์ที่ครอบครอง)

**คืออะไร**: ความกว้างของช่องสัญญาณที่สถานีวิทยุ FM ใช้จริงในการแพร่คลื่น วัดจากสเปกตรัมของสัญญาณที่ส่งออกอากาศ

**ทำไมต้องตรวจ**: ถ้าสถานีแพร่คลื่นกว้างเกินกว่าที่ได้รับอนุญาต จะไปรบกวนสถานีข้างเคียงที่อยู่ในความถี่ใกล้กัน

**มาตรฐาน**:

- วิทยุ FM มี Necessary Bandwidth = **180 kHz** (สัญญาณ Stereo + RDS)
- Occupied Bandwidth ต้องไม่เกินค่า Necessary Bandwidth ที่กำหนด

**วิธีวัดตาม ITU-R SM.443** — มี 2 วิธี:

**วิธีที่ 1: 99% Power Bandwidth (วิธีที่ใช้จริงในภาพตัวอย่าง)**

วัดโดยหาช่วงความถี่ที่ **กำลังส่งรวม 99%** ของกำลังทั้งหมดอยู่ภายใน โดยตัดกำลังส่งส่วนที่เหลือ 0.5% ทางซ้ายและ 0.5% ทางขวาออก

```
         ▲ Power
         │
         │    ┌──────────┐
         │   ╱ ██████████ ╲
         │  ╱ ████████████ ╲         ██ = 99% ของกำลังทั้งหมด
         │ ╱ ██████████████ ╲
         │╱ ████████████████ ╲
         └────────────────────────► Frequency
         0.5%◄─ 99% Power ──►0.5%
              ◄─ Occupied BW ──►
```

- เครื่อง Spectrum Analyzer จะรวมกำลังทั้งหมดใน Channel BW แล้วหาจุดที่ตัด 0.5% ทั้งสองฝั่ง
- เหมาะสำหรับสัญญาณ FM เพราะวัดกำลังจริงที่ส่งออกอากาศ

**วิธีที่ 2: x dB Method**

วัดจากจุดสูงสุดของสเปกตรัม แล้วหาจุดที่สัญญาณลดลง x dB ทั้งสองฝั่ง (x = 26 dB สำหรับ FM)

```
         ▲ Power
         │
    0 dB │    ┌──────────┐
         │   ╱            ╲
  -26 dB │──╱──────────────╲──  ← จุดวัด
         │ ╱                ╲
         │╱                  ╲
         └────────────────────────► Frequency
           ◄─ Occupied BW ──►
```

!!! note "ทั้งสองวิธีให้ผลใกล้เคียงกัน"

    ตาม ITU-R SM.443 ค่า x dB ถูกเลือกมาให้ผลลัพธ์เท่ากับ 99% power bandwidth สำหรับแต่ละประเภทการ modulate

**ตัวอย่างจากเครื่องจริง** — วัดสถานี 104.25 MHz ด้วย **วิธี 99% Power** ได้ OBW = 145.714 kHz (ผ่านเกณฑ์ 200 kHz):

![Occupied Bandwidth measurement](../assets/ocb.png)

จากภาพ: Channel BW = 200 kHz, % Pwr BW = 99.0% → เครื่องคำนวณว่า 99% ของกำลังอยู่ใน **145.714 kHz** ซึ่งน้อยกว่า 180 kHz = **ผ่าน**

!!! tip "ตั้งค่า Spectrum Analyzer"

    | พารามิเตอร์ | ค่าที่แนะนำ | เหตุผล |
    |------------|------------|--------|
    | **Center Freq** | ความถี่กลางของสถานี (เช่น 100.5 MHz) | ตั้งตรงกลางสัญญาณ |
    | **Span** | 360 kHz - 1 MHz | กว้างพอให้เห็นขอบสัญญาณทั้งสองฝั่ง |
    | **RBW** | 10 kHz | ละเอียดพอให้เห็นรูปร่างสเปกตรัม |
    | **VBW** | 1 - 3 เท่า RBW | ลด noise ให้อ่านค่าได้ชัด |
    | **Detector** | Sample หรือ RMS แล้วแต่| สำหรับวัด Occupied BW |
    | **Trace** | Max Hold (3-5 sweeps) | จับค่าสูงสุดให้ครบทุก modulation |

## 2. Frequency Deviation (การเบี่ยงเบนความถี่)

**คืออะไร**: ค่าที่ความถี่ของสัญญาณ FM เบี่ยงเบนออกจากความถี่กลาง (carrier) เมื่อมีการ modulate เสียง ยิ่งเสียงดัง ค่า deviation ยิ่งมาก

**ทำไมต้องตรวจ**: ถ้า deviation สูงเกินไป สัญญาณจะกว้างเกินและรบกวนช่องข้างเคียง ถ้าน้อยเกินไป เสียงจะเบาและคุณภาพไม่ดี

**มาตรฐาน**:

- **Maximum Frequency Deviation**: +/- 75 kHz สำหรับ FM Broadcasting
- ค่า Peak Deviation ต้องไม่เกิน +/- 75 kHz ในทุกสภาวะ

### ทำไมต้อง ±75 kHz — ที่มาของค่านี้

ค่า ±75 kHz ไม่ได้มาจากการสุ่มเลือก แต่คำนวณจาก **Carson's Rule**:

```
Necessary BW = 2 × (Δf + f_audio)
```

| ตัวแปร | ค่า | ความหมาย |
|--------|-----|---------|
| **Δf** | 75 kHz | Maximum frequency deviation |
| **f_audio** | 15 kHz | ความถี่เสียงสูงสุดที่ FM ส่ง |
| **Necessary BW** | 2 × (75 + 15) = **180 kHz** | แบนด์วิดท์ที่จำเป็นของ FM |

**เปรียบเทียบ**: deviation คือ **ขนาดการแกว่ง** ของสัญญาณ FM — เสียงดัง → แกว่งมาก / เสียงเบา → แกว่งน้อย

```
ความถี่
  ▲
  │                         ← ถ้า deviation > 75 kHz
  │  ┌─────────────────┐      สัญญาณจะล้ำเข้าช่องข้างเคียง!
  │  │  Channel ข้างเคียง │
  │  └─────────────────┘
  │         ▲
+75 kHz ────│─────────────── ← ขีดจำกัดบน
  │     ╱╲  │  ╱╲
  │    ╱  ╲ │ ╱  ╲
Carrier ──╱────╲╱────╲──── ← ความถี่กลาง
  │  ╱    │       ╲
  │ ╱     │        ╲
-75 kHz ──│─────────────── ← ขีดจำกัดล่าง
  │  ┌────│────────────┐
  │  │  Channel ข้างเคียง │
  │  └─────────────────┘
  │
  └──────────────────────► เวลา
```

### ความสัมพันธ์กับ Channel Spacing

FM ไทยใช้ **channel spacing = 250 kHz** ดูว่าพอดีกันอย่างไร:

```
     ◄──── 250 kHz ────►◄──── 250 kHz ────►
     │                  │                  │
     │   ◄─ 180 kHz ─►  │   ◄─ 180 kHz ─►  │
     │   ┌────────────┐ │   ┌────────────┐ │
     │   │  สถานี A   │ │   │  สถานี B   │ │
     │   └────────────┘ │   └────────────┘ │
     │  ▲  35 kHz  ▲  35 kHz  ▲            │
     │  │  guard   │  guard   │            │
     │  │  band    │  band    │            │

     ช่องว่าง 2 ข้าง = 250 - 180 = 70 kHz (ข้างละ 35 kHz)
```

| ค่า | ที่มา |
|-----|------|
| **±75 kHz** deviation | ทำให้สัญญาณกว้างได้สูงสุด 180 kHz (Carson's Rule) |
| **180 kHz** necessary BW | พอดีอยู่ใน channel spacing 250 kHz |
| **35 kHz** guard band ต่อข้าง | ป้องกันสัญญาณล้ำช่องข้างเคียง |

!!! warning "ถ้า deviation เกิน ±75 kHz"

    - Necessary BW > 180 kHz → **กินเข้าไปใน guard band**
    - ถ้าเกินมาก → **ล้ำเข้าช่องสถานีข้างเคียง** → เกิด adjacent channel interference
    - ตัวอย่าง: deviation ±100 kHz → BW = 2 × (100+15) = 230 kHz → เหลือ guard band แค่ 10 kHz ต่อข้าง

**วิธีวัดตาม ITU-R SM.1268**:

```
         ▲ Frequency
         │
+75 kHz  │─ ─ ─ ─ ─ ─ ─ ─ ─ ─  ← ขีดจำกัดสูงสุด
         │      ╱╲    ╱╲
Carrier  │────╱──╲──╱──╲────── ← ความถี่กลาง
         │  ╱      ╲╱
-75 kHz  │─ ─ ─ ─ ─ ─ ─ ─ ─ ─  ← ขีดจำกัดต่ำสุด
         │
         └──────────────────────► Time
```

**ตัวอย่างจากเครื่องจริง** — วัด Frequency Deviation สถานี 104.25 MHz ตามเกณฑ์ SM.1268-2 (PASS):

**สิ่งที่เห็นจากภาพ**:

- **เส้นแดง (R)** คือ limit line SM.1268-2 — รูปทราปีซอยด์ที่มี **แท่นบน (flat top) แบน** ที่ -19 dBm กินพื้นที่กว้าง แล้ว **ไหลลงเป็นขั้น** ทั้งสองข้างจนถึง ~-69 dBm ที่ขอบ span
- **เส้นเหลือง (Y)** คือสัญญาณ FM จริง — รูปทราปีซอยด์เช่นกัน แต่ **แคบกว่าแดง** — แท่นบนอยู่ที่ ~-19 ถึง -25 dBm มี ripple เล็กน้อย (จาก modulation), ขอบลาดลงชันกว่าเส้นแดง
- **เส้นเหลืองอยู่ใต้เส้นแดงทุกจุด** — ไม่มีจุดไหนทะลุ → เครื่องแจ้ง **PASS** (กล่องเขียวมุมขวาบน)
- **Marker M1** อยู่บนแท่นบนของ yellow ที่ 104.25 MHz อ่านค่า -10.0 dBm (peak จาก Max Hold)
- **Noise floor** ประมาณ -65 ถึง -69 dBm ที่ขอบ span

**ตีความผลวัด**:

1. **Carrier peak** = -10 dBm ที่ 104.25 MHz (ตรง marker M1)
2. **Signal ด้านบน (yellow)** — มีรูปร่าง **ทราปีซอยด์แบน** (flat-top trapezoid) กว้างประมาณ ±75 kHz ที่ระดับ -19 dBm — นี่คือ **Necessary Bandwidth** ของ FM ที่โมดูเลตเต็มที่
3. **Sides สโลปลง** อย่างชัดเจน — สัญญาณลดต่ำลงนอกช่วง ±75 kHz เป็นไปตาม Carson's Rule
4. **Limit line (red) SM.1268-2** — เป็นกรอบบนที่ครอบสัญญาณ รูปทราปีซอยด์เช่นกัน แต่กว้างกว่าสัญญาณจริง
5. **สัญญาณทุกจุดอยู่ใต้ limit line** → เครื่องแจ้ง **PASS** (กล่องเขียวมุมขวาบน)
6. **Noise floor** ประมาณ -70 ถึง -75 dBm (RBW 10 kHz) — มี dynamic range เหลือ ~60 dB จาก carrier

!!! note "สิ่งที่สังเกตได้จากภาพนี้"

    - **รูปร่างทราปีซอยด์** = สัญญาณ FM ที่โมดูเลตอย่างถูกต้อง — ไม่ใช่ carrier เปล่า ไม่ใช่สัญญาณเบี่ยงเบนผิดปกติ
    - **ความกว้าง -3 dB ≈ 150 kHz** → Deviation peak ≈ ±75 kHz (ตรงตามมาตรฐาน)
    - **สัญญาณไม่ล้ำออกนอกช่องข้างเคียง** (นอก ±100 kHz สัญญาณลดลงอย่างรวดเร็ว)
    - **ไม่มี spurious spike** นอกกรอบ mask

    | **Span** | 200-360 kHz | เห็น deviation ทั้งสองฝั่ง |

### Frequency Deviation Mask ตามมาตรฐาน กสทช. มท. ๘๐๐๑-๒๕๕๕

มาตรฐาน กสทช. กำหนด **Spectrum Mask สำหรับตรวจ Frequency Deviation** ไว้ในข้อ 2.3 — สัญญาณ FM ต้องอยู่ภายใน mask นี้

#### พิกัด Mask (สมมาตรซ้ายขวา)

| Offset จาก carrier (kHz) | ระดับ (dBc) | ความหมาย |
|--------------------------|-----------|---------|
| **-500** | **-85** | ห่างจาก carrier 500 kHz → ต้องต่ำกว่า carrier 85 dB (แทบไม่มีสัญญาณ) |
| **-300** | **-85** | ยังอยู่ในโซนเข้มงวด — ห้ามมีพลังงานรั่วไหล |
| **-200** | **-80** | เริ่มผ่อนปรนเล็กน้อย (-80 แทน -85) เพราะใกล้ขอบ channel |
| **-100** | **0** | อยู่ใน channel ปกติ — สัญญาณเต็มกำลังได้ |
| **+100** | **0** | อยู่ใน channel ปกติ — สัญญาณเต็มกำลังได้ |
| **+200** | **-80** | ขอบ channel ฝั่งขวา — เริ่มเข้มงวด |
| **+300** | **-85** | นอก channel — ต้องต่ำมาก |
| **+500** | **-85** | ไกลจาก carrier — ต้องแทบไม่มีสัญญาณ |

!!! note "อ่านตารางอย่างไร"

    **Offset** = ระยะห่างจากความถี่ carrier (ความถี่กลางของสถานี)

    - ค่า **ลบ (-)** = ด้านซ้ายของ carrier (ความถี่ต่ำกว่า)
    - ค่า **บวก (+)** = ด้านขวาของ carrier (ความถี่สูงกว่า)

    **dBc** = dB relative to carrier — เปรียบเทียบกับกำลังของ carrier

    - **0 dBc** = เท่ากับ carrier (สัญญาณเต็มกำลัง)
    - **-80 dBc** = ต่ำกว่า carrier 80 dB = กำลัง **น้อยกว่า 100 ล้านเท่า**
    - **-85 dBc** = ต่ำกว่า carrier 85 dB = กำลัง **น้อยกว่า 316 ล้านเท่า**

    ตัวอย่าง: สถานี carrier power = 500 W

    | ระดับ | กำลังจริง | เปรียบเทียบ |
    |------|---------|-----------|
    | 0 dBc | 500 W | เท่ากับ carrier |
    | -80 dBc | 0.000005 W (5 μW) | น้อยมากจนแทบวัดไม่ได้ |
    | -85 dBc | 0.0000016 W (1.6 μW) | น้อยกว่า -80 อีก ~3 เท่า |

#### แปลง dBc → dBm สำหรับใช้งานในสนาม

!!! warning "เครื่อง Spectrum Analyzer วัดเป็น dBm ไม่ใช่ dBc"

    Mask กสทช. กำหนดเป็น **dBc** (เทียบกับ carrier) แต่เครื่องในสนามแสดงเป็น **dBm** (กำลังสัมบูรณ์) — ต้อง **แปลงก่อน** ถึงจะตั้ง limit line ได้

**สูตรแปลง**:

```
Level (dBm) = Carrier Power (dBm) + Offset (dBc)
```

เนื่องจาก dBc เป็นค่าลบ (เช่น -85) การ "บวก" จึงได้ค่าที่ต่ำลง

**ขั้นตอนในสนาม**:

```
1.  วัด Carrier Power ของสถานีก่อน (Peak ที่ความถี่กลาง)
         │
         ├→ ตั้ง Center Freq = ความถี่สถานี
         ├→ Span = 500 kHz, RBW = 10 kHz
         ├→ Trace = Max Hold
         └→ Marker → Peak → อ่านค่า (เช่น -20 dBm)
         │
2.  แปลงทุกจุดของ mask:
         │
         └→ dBm = Carrier + dBc
         │
3.  ใส่ค่า dBm ที่แปลงแล้วลงใน Limit Line บนเครื่อง
```

!!! example "ตัวอย่าง: สถานี 100.0 MHz, Carrier = -20 dBm"

    | จุด | Offset (kHz) | Mask (dBc) | Level (dBm) = -20 + dBc |
    |-----|-------------|-----------|------------------------|
    | 1 | -500 | -85 | **-105 dBm** |
    | 2 | -300 | -85 | **-105 dBm** |
    | 3 | -200 | -80 | **-100 dBm** |
    | 4 | -100 | 0   | **-20 dBm** |
    | 5 | +100 | 0   | **-20 dBm** |
    | 6 | +200 | -80 | **-100 dBm** |
    | 7 | +300 | -85 | **-105 dBm** |
    | 8 | +500 | -85 | **-105 dBm** |

    → นำค่า **dBm** คอลัมน์ขวาไปใส่ใน Limit Line ของเครื่อง FSH8

!!! tip "ข้อควรระวัง"

    - **Carrier ต้องวัดให้ถูก** — ใช้ Marker Peak และ Max Hold เสมอ ไม่งั้นค่าแปลงจะผิดทั้งหมด
    - **RBW ต้องเท่ากันตอนวัด carrier กับตอนวัด mask** — ถ้าเปลี่ยน RBW ค่าที่อ่านจะเปลี่ยนด้วย
    - **ถ้าใช้ attenuator/preamp** ต้องเปิดไว้เหมือนกันทั้งสองครั้ง
    - **เปลี่ยนสถานี → ต้องวัด carrier ใหม่** และคำนวณ mask ใหม่ เพราะกำลังส่งแต่ละสถานีไม่เท่ากัน

#### ระดับ Carrier ที่เหมาะสมสำหรับวัด Mask

**ช่วงที่ดีที่สุด: Carrier = -30 ถึง -10 dBm** ที่อินพุตเครื่อง Spectrum Analyzer

```
  ┌─────────────────────────────────────────────┐
  │  +10 dBm  ▲  เสี่ยง overload / compression   │  ← สูงไป
  │           │                                  │
  │   0 dBm  ─┼─ 1 dB compression (เครื่องเพี้ยน)│
  │           │                                  │
  │ -10 dBm  ─┼─ ◄── ดีที่สุด  ✓                 │
  │ -20 dBm  ─┼─ ◄── ดีที่สุด  ✓                 │  sweet spot
  │ -30 dBm  ─┼─ ◄── พอใช้ได้                    │
  │           │                                  │
  │ -40 dBm  ─┼─ เริ่มเห็น -85 dBc ไม่ชัด        │
  │           │                                  │
  │-110 dBm  ─┴─ Noise floor (RBW 10 kHz)        │  ← ต่ำไป
  └─────────────────────────────────────────────┘
```

**เหตุผล**: ต้องพอดีระหว่าง 2 ข้อจำกัด

1. **ไม่สูงเกินไป** → ไม่ให้เครื่อง overload / compression (ทำให้ค่าเพี้ยน)
2. **ไม่ต่ำเกินไป** → ต้องเห็นจุด -85 dBc ของ mask เหนือ noise floor อย่างน้อย 10 dB

#### ตรวจสอบว่าเห็น -85 dBc ได้หรือไม่

Mask จุดต่ำสุด = **-85 dBc** → จุดนี้ต้องอยู่เหนือ noise floor ≥ 10 dB

| Carrier (dBm) | จุด -85 dBc (dBm) | Noise Floor | เห็นไหม |
|---------------|------------------|-------------|---------|
| **-10** | -95 | -110 | ✓ ชัดเจน (เหนือ noise 15 dB) |
| **-15** | -100 | -110 | ✓ ดี (เหนือ noise 10 dB) |
| **-20** | -105 | -110 | ✓ พอเห็น (เหนือ noise 5 dB) |
| **-30** | -115 | -110 | ✗ จมใน noise |
| **-40** | -125 | -110 | ✗ มองไม่เห็นเลย |

!!! example "ตัวอย่าง: วัดสถานี 100.0 MHz ที่ carrier = -15 dBm"

    **ขั้นตอน**:

    ```
    1. ตั้งเครื่อง:
         Center Freq     = 100.0 MHz
         Span            = 500 kHz
         RBW             = 10 kHz

         Detector        = Peak

    | จุด | Freq (MHz) | dBc | dBm (= -15 + dBc) |

    | 7 | 100.300 | -85 | **-100** |
    | 8 | 100.500 | -85 | **-100** |

    **ผลลัพธ์ที่คาดหวัง**:

!!! tip "วิธีปรับระดับในสนาม"

    **Carrier แรงเกินไป** (> -5 dBm):

    - เปิด **Attenuator** (+10 ถึง +20 dB)
    - ขยับเสาให้ห่างจากสถานีมากขึ้น

    **Carrier อ่อนเกินไป** (< -30 dBm):

    - เข้าใกล้สถานีขึ้น
    - ใช้เสาอากาศ gain สูงขึ้น
    - เปิด **Preamp** (ถ้ามี) — แต่ระวัง overload ถ้ามีสถานีแรงอยู่ใกล้
    - ลด **RBW** ลง (10 kHz → 3 kHz → 1 kHz) — ทุกครั้งที่ลด 10 เท่า noise floor ลด ~10 dB

#### ทำไม mask เป็นรูปนี้

```
                 ◄── ช่องข้างเคียง ──►◄──── ช่องของสถานี ────►◄── ช่องข้างเคียง ──►
                 │                    │                       │                    │
  สัญญาณ        │  ห้ามรั่วมาที่นี่!   │   อยู่ตรงนี้ได้เต็มที่   │  ห้ามรั่วไปที่นี่!   │
  ของสถานี       │  (-85 dBc)         │   (0 dBc)              │  (-85 dBc)         │
  อื่น          │                    │        ╱╲              │                    │
                │                    │       ╱  ╲             │                    │
                │                    │      ╱    ╲            │                    │

- **±100 kHz (0 dBc)**: ช่วงนี้คือ **Necessary Bandwidth** ของ FM = สัญญาณอยู่ได้เต็มที่
- **±100 → ±200 kHz (0 → -80 dBc)**: ขอบ channel → สัญญาณต้อง **ลดลงอย่างรวดเร็ว** ไม่ให้ล้ำช่องข้างเคียง
- **±200 → ±300 kHz (-80 → -85 dBc)**: ลดอีกเล็กน้อย → เข้าสู่ **guard band** ระหว่างช่อง
- **±300 kHz ขึ้นไป (-85 dBc)**: อยู่ใน **ช่องสัญญาณของสถานีอื่น** แล้ว → ต้องแทบไม่มีพลังงานเหลือ

## 3. Spurious Emission (สัญญาณแพร่ปลอม)

**คืออะไร**: สัญญาณที่แพร่ออกมานอกช่องสัญญาณที่ได้รับอนุญาต เช่น สัญญาณฮาร์มอนิก (harmonic) ที่เป็นทวีคูณของความถี่หลัก หรือสัญญาณรบกวนอื่นๆ

**ทำไมต้องตรวจ**: Spurious emission จะไปรบกวนบริการวิทยุอื่นที่อยู่คนละย่านความถี่

**ตัวอย่าง**: สถานี FM ที่ 100 MHz อาจมี harmonic ที่ 200 MHz (2nd), 300 MHz (3rd) ซึ่งจะไปรบกวนย่านอื่น

```
    ▲ Power
    │
    │   ┌┐                     สัญญาณหลัก (100 MHz)
    │   ││
    │   ││
    │   ││          ┌┐         2nd Harmonic (200 MHz) ← ต้องต่ำกว่าเกณฑ์
    │   ││          ││  ┌┐     3rd Harmonic (300 MHz) ← ต้องต่ำกว่าเกณฑ์
    │   ││          ││  ││
    └───┴┴──────────┴┴──┴┴───────► Frequency
       100         200  300 MHz
```

**ตัวอย่างจากเครื่องจริง** — สแกนย่าน 87-137 MHz เพื่อดู Unwanted Emission ของสถานี 104.25 MHz (M1) พร้อมตรวจย่านการบิน 108-137 MHz (M2-M3):

![Unwanted Emission scan 87-137 MHz](../assets/unwated.png)

จากภาพ: M1 (104.25 MHz) = -4.4 dBm, M2 (108 MHz) = -92.9 dBm, M3 (137 MHz) = -91.9 dBm — ไม่พบสัญญาณรบกวนในย่านการบิน


!!! tip "ตั้งค่า Spectrum Analyzer"

    | พารามิเตอร์ | ค่าที่แนะนำ | เหตุผล |
    |------------|------------|--------|
    | **Center Freq** | ความถี่ของ harmonic ที่ต้องการวัด | เช่น 200 MHz สำหรับ 2nd harmonic |
    | **Span** | กว้าง เช่น 500 MHz - 1 GHz | เพื่อหา spurious ทุกตำแหน่ง |
    | **RBW** | 10-100 kHz | กว้างพอจับ spurious ที่อาจกระจาย |
    | **VBW** | เท่ากับ RBW | ความเร็ว sweep เหมาะสม |
    | **Ref Level** | ตั้งให้สัญญาณหลักอยู่ใกล้ top | ใช้ dynamic range สูงสุด |
    | **Attenuation** | Auto หรือ 10-20 dB | ป้องกัน overload จากสัญญาณหลักที่แรง |

    **ขั้นตอน:**

    1. วัดระดับสัญญาณหลัก (fundamental) ก่อน
    2. เปลี่ยน center freq ไปที่ตำแหน่ง harmonic (2x, 3x, 4x, 5x)
    3. วัดระดับ spurious แต่ละตัว
    4. คำนวณผลต่าง (dBc) = ระดับสัญญาณหลัก - ระดับ spurious
    5. เปรียบเทียบกับเกณฑ์มาตรฐาน

## 4. Intermodulation ที่ทำให้เกิด Spurious ในย่าน 108-137 MHz (ย่านการบิน)

**คืออะไร**: เมื่อสัญญาณจากสถานี FM หลายสถานีที่อยู่ใกล้กัน (บนเสาเดียวกันหรือพื้นที่ใกล้เคียง) ผสมกันในอุปกรณ์ที่ไม่เป็นเชิงเส้น (non-linear) เช่น เครื่องส่ง, สายอากาศ, หรือแม้แต่ชิ้นส่วนโลหะที่เป็นสนิม จะเกิดสัญญาณใหม่ที่ความถี่อื่น เรียกว่า **Intermodulation Product (IM)**

!!! danger "ทำไมอันตราย"

    ย่าน **108-137 MHz** เป็นย่านการบิน (Aeronautical Band) ใช้สำหรับระบบช่วยการเดินอากาศ เช่น ILS, VOR, และวิทยุสื่อสารการบิน หาก IM จากสถานี FM ตกในย่านนี้ อาจรบกวนการนำร่องเครื่องบินได้

### สูตรคำนวณ Intermodulation

**3rd Order Intermodulation** (อันตรายที่สุด เพราะมีกำลังสูงและใกล้ย่านเดิม):

เมื่อมีสถานี 2 สถานีที่ความถี่ **f1** และ **f2**:

```
IM3 = 2f1 - f2    หรือ    IM3 = 2f2 - f1
```

### ตัวอย่างจริง: FM สร้าง IM ตกในย่านการบิน

!!! example "ตัวอย่างที่ 1: สถานี 2 สถานีบนเสาเดียวกัน"

    สถานี A: **f1 = 104.5 MHz**
    สถานี B: **f2 = 98.0 MHz**

    ```
    IM3 = 2(104.5) - 98.0 = 209.0 - 98.0 = 111.0 MHz  ← ตกในย่านการบิน!
    IM3 = 2(98.0) - 104.5 = 196.0 - 104.5 = 91.5 MHz   ← ยังอยู่ในย่าน FM
    ```

    **111.0 MHz** อยู่ในย่าน VOR/ILS ของการบิน — อันตรายมาก!

!!! example "ตัวอย่างที่ 2: สถานี 3 สถานีใกล้กัน"

    สถานี A: **f1 = 100.5 MHz**
    สถานี B: **f2 = 104.0 MHz**
    สถานี C: **f3 = 107.5 MHz**

    ```
    IM3 = 2(104.0) - 100.5 = 107.5 MHz  ← ซ้อนกับสถานี C
    IM3 = 2(107.5) - 104.0 = 111.0 MHz  ← ตกในย่านการบิน!
    IM3 = 2(107.5) - 100.5 = 114.5 MHz  ← ตกในย่านการบิน!

    IM3 จาก 3 ความถี่: f1 + f2 - f3 = 100.5 + 104.0 - 107.5 = 97.0 MHz
    IM3 จาก 3 ความถี่: f2 + f3 - f1 = 104.0 + 107.5 - 100.5 = 111.0 MHz ← การบิน!
    ```

!!! example "ตัวอย่างที่ 3: คำนวณย้อนกลับ — หาว่าคู่ไหนสร้างปัญหา"

    พบสัญญาณรบกวนที่ **118.1 MHz** (ความถี่สื่อสารหอบังคับการบิน)

    หาคู่สถานี FM ที่อาจเป็นต้นเหตุ:

    ```
    IM3 = 2f1 - f2 = 118.1
    ถ้า f1 = 106.0 → f2 = 2(106.0) - 118.1 = 93.9 MHz

    ตรวจสอบ: มีสถานี FM ที่ 106.0 และ 93.9 MHz หรือไม่?
    ถ้ามี → นี่คือต้นเหตุ!
    ```

### วิธีตรวจสอบ IM ในย่าน 108-137 MHz

!!! tip "ตั้งค่า Spectrum Analyzer"

    | พารามิเตอร์ | ค่าที่แนะนำ | เหตุผล |
    |------------|------------|--------|
    | **Start Freq** | 108 MHz | เริ่มต้นย่านการบิน |
    | **Stop Freq** | 137 MHz | สิ้นสุดย่านการบิน |
    | **RBW** | 10 kHz | ละเอียดพอจับ IM ที่อาจเบา |
    | **VBW** | 1-3 เท่า RBW | ลด noise floor |
    | **Ref Level** | -20 ถึง -40 dBm | IM มักมีกำลังต่ำ ต้องตั้ง ref ให้ต่ำ |
    | **Attenuation** | 0 dB (ถ้าไม่มีสัญญาณแรง) | เพิ่มความไวในการวัด |
    | **Trace** | Max Peak |
    | **Trace** | Max Hold (หลาย sweep) | จับ IM ที่อาจเกิดเป็นช่วงๆ |

    **ขั้นตอน:**

    1. ตั้ง Spectrum Analyzer ที่ย่าน 108-137 MHz
    2. สแกนหาสัญญาณที่ไม่ควรมี (ย่านนี้ควรมีแค่สัญญาณการบิน)
    3. ถ้าพบสัญญาณแปลก ให้จดความถี่ไว้
    4. คำนวณย้อนกลับว่าคู่สถานี FM ใดที่อาจสร้าง IM ที่ความถี่นั้น
    5. ตรวจสอบที่ตัวเครื่องส่งและสายอากาศของสถานีต้นเหตุ

### ประเภทของ Intermodulation — Type A vs Type B

IM แบ่งตามจุดที่เกิดปัญหาเป็น 2 ประเภท:

```
Type A: เกิดที่ "เครื่องส่ง" ของสถานีต้นเหตุ
═══════════════════════════════════════════

  สถานี A (f1)        สถานี B (f2)
  ┌──────────┐        ┌──────────┐
  │ เครื่องส่ง │        │ เครื่องส่ง │
  └────┬─────┘        └────┬─────┘
       │                   │
       ▼                   │
  ┌─────────┐              │
  │สายอากาศA│◄─────────────┘  สัญญาณ f2 ย้อนเข้ามา
  └────┬────┘                 ทางสายอากาศของสถานี A
       │
       ▼
  IM เกิดใน PA ของสถานี A (non-linear)
  แล้วส่งออกอากาศไปพร้อมกับ f1


Type B: เกิดที่ "ภายนอก" ระหว่างสายอากาศ
═══════════════════════════════════════════

  สถานี A (f1)        สถานี B (f2)
  ┌──────────┐        ┌──────────┐
  │ เครื่องส่ง │        │ เครื่องส่ง │
  └────┬─────┘        └────┬─────┘
       │                   │
       ▼                   ▼
  ┌─────────┐         ┌─────────┐
  │สายอากาศA│         │สายอากาศB│
  └────┬────┘         └────┬────┘
       │                   │
       └───────┐  ┌────────┘
               ▼  ▼
          ┌──────────┐
          │ จุดผสม    │  ← สนิม, โลหะสัมผัสกัน,
          │ (non-     │    connector เสื่อม
          │  linear)  │
          └────┬─────┘
               ▼
          IM แผ่ออกอากาศจากจุดนี้
```

### เปรียบเทียบ Type A vs Type B

| | **Type A** (Transmitter IM) | **Type B** (External IM) |
|--|---------------------------|------------------------|
| **เกิดที่** | ภายในเครื่องส่ง/PA ของสถานีใดสถานีหนึ่ง | ภายนอก — จุดต่อ, สนิม, โครงสร้างโลหะ |
| **สาเหตุ** | สัญญาณจากสถานีอื่น **ย้อนเข้า** ทางสายอากาศเข้าไปผสมใน PA | สัญญาณ 2+ สถานี **ผสมกัน** ที่จุดสัมผัสโลหะภายนอก |
| **เปรียบเทียบ** | เหมือน **คนอื่นตะโกนเข้ามาในไมค์ของเรา** แล้วลำโพงเราส่งเสียงผสมออกไป | เหมือน **เสียง 2 คนกระทบกำแพง** แล้วเกิดเสียงสะท้อนแปลกๆ |
| **กำลังของ IM** | มักแรงกว่า (เพราะผ่าน PA ที่ขยายสัญญาณ) | มักเบากว่า (ไม่มีตัวขยาย) |
| **ตรวจพบ** | วัดที่ output ของเครื่องส่งสถานีต้นเหตุ (conducted) | วัดในอากาศ (radiated) — วัดที่เครื่องส่งไม่เจอ |
| **แก้ไข** | ใส่ **Isolator/Circulator** หรือ **Band-pass Filter** ที่ output เครื่องส่ง | แก้ไข **จุดสัมผัสโลหะ**, เปลี่ยน connector, เพิ่มระยะห่างสายอากาศ |

!!! example "ตัวอย่าง Type A"

    สถานี A (104.5 MHz) และสถานี B (98.0 MHz) อยู่บนเสาเดียวกัน

    สัญญาณ 98.0 MHz จากสถานี B ย้อนเข้าสายอากาศสถานี A → เข้า PA ของสถานี A ที่ทำงานแบบ non-linear → เกิด IM3 = 2(104.5) − 98.0 = **111.0 MHz** → PA ขยายแล้วส่งออกอากาศ

    **วิธีตรวจ:** วัด conducted spurious ที่ output ของเครื่องส่งสถานี A จะเจอสัญญาณ 111.0 MHz

    **วิธีแก้:** ใส่ Isolator ที่ output เครื่องส่งสถานี A เพื่อป้องกันสัญญาณย้อนเข้า

!!! example "ตัวอย่าง Type B"

    สถานี A (104.5 MHz) และสถานี B (98.0 MHz) อยู่บนเสาเดียวกัน

    ทั้งสองสัญญาณแผ่ออกอากาศปกติ แต่ไปผสมกันที่ **รอยต่อโลหะที่เป็นสนิม** บนโครงเสา (Rusty Bolt Effect) → เกิด IM3 = **111.0 MHz** แผ่จากจุดนั้น

    **วิธีตรวจ:** วัด conducted ที่เครื่องส่งทั้ง 2 สถานี → **ไม่เจอ** IM ต้องวัดแบบ radiated (ในอากาศ) ถึงจะเจอ

    **วิธีแก้:** ตรวจหาจุดสัมผัสโลหะที่เป็นสนิมบนเสา ทำความสะอาดหรือเปลี่ยน

### วิธีแยก Type A จาก Type B — ใช้ Notch Filter

**Notch Filter** (ตัวกรองตัดความถี่) คือเครื่องมือสำคัญในการแยก Type A จาก Type B ในภาคสนาม

**หลักการ:** ใส่ Notch Filter ตัดสัญญาณของสถานีใดสถานีหนึ่งออก แล้วดูว่า IM หายไปหรือไม่

```
ทดสอบ: ใส่ Notch Filter ที่ output สถานี A (ตัด f1 ออก)

  ก่อนใส่ Notch                    หลังใส่ Notch ที่สถานี A
  ┌──────────┐                     ┌──────────┐
  │ สถานี A  │──► f1 ออกอากาศ      │ สถานี A  │──► f1 ออกอากาศ
  └──────────┘                     └──┬───────┘
  ┌──────────┐                        │ Notch Filter (ตัด f1)
  │ สถานี B  │──► f2 ออกอากาศ         ▼
  └──────────┘                     สัญญาณ f1 ไม่ย้อนเข้า PA สถานี B
                                   
  IM = 111 MHz ✓ (มี)             IM = ?
```

**ผลลัพธ์:**

| ใส่ Notch ที่ | IM หายไป? | แปลว่า |
|-------------|----------|--------|
| output สถานี A | IM หาย | **Type A** — IM เกิดจากสัญญาณ f1 ย้อนเข้า PA สถานี B |
| output สถานี B | IM หาย | **Type A** — IM เกิดจากสัญญาณ f2 ย้อนเข้า PA สถานี A |
| output ทั้ง 2 สถานี | IM **ยังอยู่** | **Type B** — IM ไม่ได้เกิดในเครื่องส่ง เกิดภายนอก |

!!! tip "สรุปวิธีใช้ Notch Filter"

    1. วัด IM ก่อนใส่ Notch → จดระดับ IM ไว้
    2. ใส่ Notch Filter ที่ output สถานี A → ดูว่า IM หายหรือลดลงหรือไม่
    3. ถอด Notch ออก แล้วใส่ที่ output สถานี B → ดูอีกครั้ง
    4. ถ้าใส่ Notch ที่ไหนแล้ว IM หาย → **Type A** ปัญหาอยู่ที่เครื่องส่งอีกฝั่ง (ฝั่งที่ไม่ได้ใส่ Notch)
    5. ถ้าใส่ Notch ทั้ง 2 ฝั่งแล้ว IM ยังอยู่ → **Type B** ต้องหาจุดผสมภายนอก

### Flowchart รวม

```
พบ IM ในย่าน 108-137 MHz
             │
             ▼
ขั้นตอนที่ 1: ใส่ Notch Filter ที่ output สถานี A
             │
             ├── IM หาย → Type A (f1 ย้อนเข้า PA สถานี B)
             │              → แก้: ใส่ Isolator ที่สถานี B
             │
             └── IM ยังอยู่
                       │
                       ▼
ขั้นตอนที่ 2: ใส่ Notch Filter ที่ output สถานี B
             │
             ├── IM หาย → Type A (f2 ย้อนเข้า PA สถานี A)
             │              → แก้: ใส่ Isolator ที่สถานี A
             │
             └── IM ยังอยู่
                       │
                       ▼
ขั้นตอนที่ 3: Type B — IM เกิดภายนอก
              → แก้: ตรวจหาจุดสนิม/โลหะสัมผัส บนเสา
```

### สาเหตุหลักที่ทำให้เกิด IM

| สาเหตุ | ประเภท | คำอธิบาย | วิธีแก้ |
|--------|-------|---------|--------|
| สัญญาณย้อนเข้าเครื่องส่ง | **Type A** | สัญญาณจากสถานีอื่นย้อนเข้า PA ผ่านสายอากาศ | ติด Isolator/Circulator ที่ output |
| PA ทำงาน non-linear | **Type A** | เครื่องส่งกำลังสูงเกิน หรือ PA เสื่อม | ลดกำลังส่ง, ปรับจูน amplifier |
| สายอากาศหลายสถานีบนเสาเดียวกัน | **A หรือ B** | สัญญาณ RF แรงๆ coupling กัน | ติด Band-pass Filter, เพิ่มระยะห่าง |
| Connector/สายนำสัญญาณเสื่อม | **Type B** | จุดต่อที่เป็นสนิมทำตัวเป็น diode | เปลี่ยน connector, ทำความสะอาด |
| สายอากาศใกล้โครงสร้างโลหะ | **Type B** | โลหะที่สัมผัสกันทำตัวเป็น mixer (Rusty Bolt Effect) | แก้ไขโครงสร้าง, เพิ่มระยะห่าง |
